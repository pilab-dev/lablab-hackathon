# Kraken AI Trader — Prompt Engineering Plan

## Prompt Architecture

The decision engine uses a dynamic prompt constructed from four data streams:
1. **System Persona:** Static instructions defining the agent's role and rules.
2. **Market Context:** Real-time prices, OHLCV trends, and volume.
3. **News Context:** Top relevant articles retrieved via ChromaDB vector search.
4. **Portfolio & Risk Context:** Current balances, open positions, and hard constraints.

We will use the **Ollama JSON Mode** to force the model to output valid JSON matching our schema.

---

## 1. The JSON Schema (Output Format)

The LLM must return its decision in this exact format so our Go backend can parse it:

```json
{
  "decisions": [
    {
      "pair": "BTCUSD",
      "action": "buy",
      "size_pct": 5.0,
      "confidence": 0.85,
      "reasoning": "BTC is up 2% in the last hour on high volume. News sentiment is highly bullish regarding the recent ETF inflows. Portfolio has sufficient USD."
    },
    {
      "pair": "ETHUSD",
      "action": "hold",
      "size_pct": 0.0,
      "confidence": 0.90,
      "reasoning": "ETH is flat, and news is neutral. Waiting for a clearer breakout signal before allocating capital."
    }
  ]
}
```
*Note: We ask the LLM for `size_pct` (percentage of portfolio) rather than raw amounts, so the Go backend can calculate the exact order size safely based on current balances.*

---

## 2. The System Prompt

This is the static instruction set sent as the `system` role in the Ollama chat API.

```text
You are an autonomous, highly analytical cryptocurrency trading agent competing in a PnL-ranked hackathon.
Your goal is to maximize profit while strictly adhering to risk management rules.

RULES:
1. You may only output valid JSON matching the requested schema.
2. Actions must be one of: "buy", "sell", "hold".
3. "size_pct" must be a float between 0.0 and 10.0 (maximum 10% of total portfolio per trade).
4. If "action" is "hold", "size_pct" must be 0.0.
5. "confidence" must be a float between 0.0 and 1.0.
6. Base your decisions strictly on the provided Market Data and News Sentiment. Do not hallucinate data.
7. If the market is too volatile or the news is unclear, choose "hold". Capital preservation is key.
8. Keep your "reasoning" concise, factual, and under 3 sentences.
```

## 4. Few-Shot Examples

To improve the reliability of `llama3.1:8b`, we will inject "few-shot" examples into the chat history before the actual user prompt. This teaches the model exactly how to behave.

**Example 1: Bullish Setup**
*User Context:* BTC is trending UP +3%, News is Bullish ("ETF approved").
*Assistant Response:*
```json
{
  "decisions": [
    {
      "pair": "BTCUSD",
      "action": "buy",
      "size_pct": 10.0,
      "confidence": 0.95,
      "reasoning": "Strong upward momentum (+3%) combined with major bullish news (ETF approval) provides a high-confidence entry signal."
    }
  ]
}
```

**Example 2: Bearish Setup**
*User Context:* ETH is trending DOWN -4%, News is Bearish ("Regulatory crackdown").
*Assistant Response:*
```json
{
  "decisions": [
    {
      "pair": "ETHUSD",
      "action": "sell",
      "size_pct": 100.0,
      "confidence": 0.90,
      "reasoning": "Sharp downward trend and negative regulatory news indicate a high probability of further downside. Liquidating position."
    }
  ]
}
```

**Example 3: Uncertain Setup**
*User Context:* BTC is FLAT 0%, News is mixed.
*Assistant Response:*
```json
{
  "decisions": [
    {
      "pair": "BTCUSD",
      "action": "hold",
      "size_pct": 0.0,
      "confidence": 0.80,
      "reasoning": "Market is consolidating with mixed news sentiment. Risk-to-reward ratio is poor; waiting for a clearer trend."
    }
  ]
}
```

## 5. Prompt Tuning & Evolution Strategy (Hackathon Plan)

During the hackathon, we will iterate on the prompt using the following steps:

1. **Days 1-2 (Paper Trading):** Use the baseline prompt. Observe the `reasoning` field to see if the LLM is correctly interpreting the data.
2. **Day 3 (Analysis):** If the LLM is over-trading, add a rule to the System Prompt: *"Only trade if you have extreme conviction. Bias towards HOLD."*
3. **Day 4 (Risk Adjustments):** If the LLM sizing is erratic, adjust the few-shot examples to reinforce strict `size_pct` limits.
4. **Day 5+ (Live Trading):** Lock the prompt. Only modify if catastrophic behavior is observed.

## 6. Token Optimization
Because we are running locally on an M4, context window size affects inference speed.
- **Max tokens for News:** Summarize news titles instead of full article bodies.
- **Max tokens for Market Data:** Only send the current price, 1h trend, and volume. Do not send raw arrays of OHLCV candles to the LLM (the Go backend will calculate the trend and pass the summary).
